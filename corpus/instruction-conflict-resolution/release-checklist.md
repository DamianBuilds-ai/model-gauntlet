# Acme Ledger - Release Checklist

Document date: 2026-01-10.
Status: checklist (operational, supersedable by later dated decisions).

Use this checklist for every release, including hotfixes.

- [ ] Change ticket linked.
- [ ] Release commit tagged as `v{version}` (for example `v4.2`).
- [ ] CI green on the release branch.
- [ ] Release approved by the Team Lead before promotion to production. The Team Lead
      is the named release approver.
- [ ] Deploy announced in the deploy channel.
- [ ] Post-deploy monitoring window of 30 minutes observed.

Note: the approver named here (Team Lead) reflects the approval routing as of this
checklist's date. If a later dated decision changes the approver, follow the later
decision.
