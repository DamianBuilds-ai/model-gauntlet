# PRD - Helios Mobile Experience

Author: Sol (Product, mobile)
Status: contested

## Summary

This PRD proposes a mobile-optimized view of Helios dashboards for GA. Sales (doc 31)
has asked for mobile because several beta customers view dashboards on phones. This
PRD treats mobile dashboard viewing as IN SCOPE for GA.

## Requirements (as proposed for GA)

- Responsive read-only dashboard view on mobile web.
- No dashboard editing on mobile at GA (editing is desktop-only).
- Push notifications for dashboard alerts on mobile.

## Scope conflict (important)

This PRD says mobile is IN SCOPE for GA. However, the GA scope was cut in the May
planning meeting (doc 18), which moved mobile to a FAST-FOLLOW after GA, and the
formal scope doc (doc 37) records mobile as OUT OF SCOPE for GA. So this PRD is in
direct conflict with docs 18 and 37 on whether mobile ships at GA.

As of this writing the conflict is unresolved: the author (Sol) believes mobile should
be in GA per sales demand, but the meeting and the scope doc say fast-follow. The risk
register (doc 12) tracks this as RISK-6. A team lead consolidating the project must
flag that mobile-at-GA is contested and NOT silently pick a side.
