# Vendor One-Pager - Strato DB

Synthetic corpus doc 3 of 16. Project one-pager for the open-source option (community
framing - emphasises control and raw performance, underplays ops cost).

## Strato DB

An open-source columnar database Riverbend would self-host. Free to license; you pay
for the infrastructure you run it on and the people who run it.

## Highlights

- Best-in-class raw query speed on analytical workloads (see the benchmarks, docs 05
  and 06).
- No license fee - only infrastructure cost.
- Full control over data location (you choose where the servers live, so EU residency
  is whatever you make it).
- Large open-source community.

## The catch the page understates

Strato DB is self-MANAGED. You run the cluster, you patch it, you handle scaling,
backups, and incident response. The ops/postmortem note (doc 08) is the honest
account of what that costs a small team. The forum thread (doc 10) is more
enthusiastic but is one community member's view, not an operational track record.

## Suited for

Teams with strong infrastructure/ops capacity who want maximum control and lowest
license cost and can afford the operational overhead. Note Riverbend's data team is
only 3 engineers - weigh that against this profile.
